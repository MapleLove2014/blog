# ZhangMapler

## 随便说说自己的专业知识和工作经历

本人于14年就读于长沙的一所普通一本学校，专业计算机科学与技术。从大四开始实习到现在3年过去了，换了几家公司，去了2个城市。主要做java后端，技术栈就是spring那一套，有小规模多线程并发爬虫和数据处理经验。熟练使用POI Excel框架，独立实现了不依靠VB脚本的Excel中的伪级联选择。Redis呢，看过核心数据结构的源码，让我收获最大的是skip list数据结构，以前也没见过。期间还闹出过一个笑话，就是C语言的Macro的纯文本替换，我还在Stackoverflow上问了这个傻X[问题](https://stackoverflow.com/questions/59559997/what-is-the-exact-syntax-or-specifications-behind-run-with-period-c-macro-in-red)。关于redis自己用的最多的就是缓存，session，分布式锁等。在处理爬虫和异步解耦时候用过MQ，对于出现的消息问题也针对处理过。有JVM故障调优经验，其实最后搞了半天，Thread Dump后发现是HttpClient的一个timeout bug。工作过程中用的最多的数据库是MySQL，略懂其中的索引，事务，锁等机制。项目中由于用的到的携程Apollo注册中心默认用的是不再维护的Eureka，自己用Zookeeper注册中心替换了下。RPC框架，用的比较多的是Dubbo。

当然上面所说的基本上都是技术点，我感觉自己在工作中最着重的是写出**漂亮**的代码。我一度觉得自己有代码洁癖，强迫症。举个最简单例子，我看到Idea工具给出的代码Warning提示，优化Tip，代码重复波浪线提示，长函数，缩进层级多等等，就走不动了，再紧的事也要等我refactor完了再说。可能是因此，我很喜欢写 `Refactor Improving the Design of Existing Code` 这本书的作者，大神 `Martin Fowler`。这本书当然没得说，经典中的经典，我还把他在公开场所的演讲都看了一遍，主要是在 `GoTo conference` 上。先不说技术，这个人真的超级有趣，吐着淡淡的英式口音（可能在美国呆久了），说话很逗，演讲的时候像个孩子一样，手舞足蹈。

有点跑远了，其实看完他的书，给我最大的感触就是，漂亮的代码不需要多么牛逼的技术，不需要日夜攻坚。纵观这本书，我没有发现啥高科技，说白了就是我们学的最基础的java那一套东西。但同样都是用java写一个功能，别人就能漂亮的干了，为什么自己写的代码就跟屎一样呢，我觉得这才是真正的差距。如果说掌握的技术是程序员的 `硬实力` ，那么把代码写漂亮就是一种 `软实力` 。自己工作工程中也会发现，很少有人会重视这一块，有的时候大家一起讨论的时候，经常会有同事说：我设计模式都看了好几遍了，就是不知道怎么重构。当然，我刚开始工作的时候也是如此，不过后来有幸遇到一位伯克利博士大神 `SJX` ，带我们做过一段时间项目。他对我们要求很高，每次 `review` 代码都会提出很多意见。刚开始我挺烦的，曾经还吵过几句，但后来发现 `真香`，也慢慢的养成了自己要写高质量代码的追求。说回来，重构和设计模式其实没有必然的关系，`Martin Fowler` 的书里面也没怎么介绍设计模式在重构里面的应用。

重构真的没有想象中的那么难，你看

1. 变量重命名

2. 方法重命名

3. 方法提取

等等这些很基础的东西都是重构。再举几个Idea里面的例子

1. Arrays.asList(A) -> Collections.singletonList(A)

2. new ArrayList() -> Collections.emptyList()

3. 直接return

    ```java
    Result result = getResult();
    return result;
    ```

    改为：

    ```java
    return getResult();
    ```

4. lock变量声明为final

等等等等。运用设计模式进行重构当然也可以，但一定要使用得当，不然可能会是本来就很难看很难维护的代码雪上加爽。本人工作中曾经对项目中的爬虫代码重构过，之前项目中的爬虫程序都是简单的复制代码，因为逻辑都差不多，最大的区别在于爬虫的网址不同。后来很自然的使用模板方法+继承重构，留一个获取网址的扩展。重构后的代码简直不要太爽，review代码时我的心情和代码红色的面积成正相关。

其实重构中，总会有同事或者leader会和我讨论性能问题，并以此为由反对我的重构。刚开始我还很有兴趣的跟他们讨论，后来我真的挺烦的，直接一句话怼死：你先去看Martin Fowler书上的性能篇，然后再来和我讨论。

关于为什么大家不怎么重视重构呢？我觉得可能有面试的因素吧，因为在面试中硬实力太重要了，没有多少人会关注你写的漂不漂亮。在工作中呢当然这也是事实，因为大家都是会分任务的，到了Deadline的时候，老板关注的永远是任务开发完了没，至于代码写的怎么样，who cares？

## 为什么来广州

这个问题，我面试的时候被问了无数次。在杭州好好的，突然就换城市。8月7号，正好一年，我差不多也走出来了。我尝试敞开心扉，不在压着。去年女朋友和我分手，原因在我。4年的感情我真的舍不得放下，在杭州浑浑噩噩的呆了一个月，决定去广州，一直到现在。我曾试图挽回，但她还是没能回心转意。来广州前，老板经常和我聊天安慰我。但我还是久久不能释怀，我终于能感觉到那种有力气没有地方使的无奈。

一年了，我现在应该放下了吧？我不知道，但我确实改变了很多，包括胖。我学会站在对方的角度去思考，这化解了我很多烦恼。比如以前我妈每天都会说我吃得少怎么会胖，我就不服气，硬着嘴说我吃了很多呀。现在回想，我发现自己真的没有必要，站在母亲的角度，她想让我吃胖一点，是为了我好，没什么问题。这样一想我的心里便能想开了。比如以前工作的时候，总会觉得产品经理的产品设计有问题。哎，现在一想他也是为了用户能够方便操作，提升用户体验，这很合理。如果自己能早点改变，可能一切都不一样了。
